# NonHumanArena 开发计划

## 一、项目概述

**项目名称**：NonHumanArena - AI辩论/游戏规则引擎  
**项目定位**：一个灵活的AI角色互动平台，支持自定义游戏规则和辩论规则，让AI角色按照预设议题展开讨论。

## 二、开发原则

1. **分阶段开发**：每个阶段可独立运行和调试
2. **核心优先**：先实现核心功能，确保系统可用
3. **接口先行**：先定义接口，再实现功能
4. **事件驱动**：模块间通过事件总线通信，降低耦合

## 五、调试策略

### 5.1 分步调试原则

1. **单元测试优先**：每个模块实现后先写单元测试
2. **Mock依赖**：使用 Mock 隔离依赖
3. **接口测试**：每个接口单独测试
4. **集成测试**：模块间集成时进行集成测试

### 5.2 调试工具

- **后端**：pytest、pdb/ipdb、SQLAlchemy echo
- **前端**：Vue DevTools、Chrome DevTools
- **WebSocket**：Postman WebSocket、在线调试工具

### 5.3 日志策略

- **开发环境**：详细日志，包含调用栈
- **生产环境**：结构化日志，重要信息
