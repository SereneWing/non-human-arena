# 场景用例

## 一、辩论场景

### 1.1 标准辩论

**场景描述**：两个AI角色（正方和反方）围绕一个辩题进行正式辩论。

**角色配置**：
- 正方辩手（Advocate）
- 反方辩手（Opponent）
- 裁判（Judge，可选）

**规则配置**：
- 轮次制发言
- 每轮限制发言字数
- 需要提供论点支撑
- 裁判评分

**事件流**：
```
1. 用户创建辩论会话，选择辩题
2. 系统初始化正方和反方AI角色
3. 正方先手发言
4. 反方针对正方论点进行反驳
5. 双方轮流发言，直到达到最大轮次或一方认输
6. 裁判根据评分标准给出评判
7. 会话结束，保存辩论记录
```

### 1.2 苏格拉底问答

**场景描述**：一个扮演老师的AI引导用户（学生）通过提问探索问题。

**角色配置**：
- 苏格拉底（导师角色）
- 学生（用户）

**规则配置**：
- 导师不能直接给出答案
- 必须通过提问引导
- 每次回应后需等待用户输入

**事件流**：
```
1. 用户提出问题或选择一个话题
2. 导师分析问题，生成第一个引导性问题
3. 用户回答或提出新问题
4. 导师根据用户回答继续提问
5. 直到用户理解或达到最大轮次
```

## 二、游戏场景

### 2.1 狼人杀

**场景描述**：多个AI角色扮演狼人杀游戏，包括狼人、村民、预言家等。

**角色配置**：
- 狼人（Werewolf）：晚上睁眼，可以杀人
- 村民（Villager）：普通角色
- 预言家（Seer）：晚上可以查验身份
- 女巫（Witch）：有救人药和毒药
- 守卫（Guardian）：可以守人
- 主持人（Moderator）：控制游戏流程

**规则配置**：
- 游戏分白天和夜晚
- 白天投票放逐
- 夜晚行动顺序
- 胜负判定条件

**事件流**：
```
夜晚阶段：
1. 主持人宣布天黑
2. 狼人睁眼，选择杀人目标
3. 预言家睁眼，选择查验目标
4. 女巫决定是否用药
5. 守卫选择守护目标
6. 主持人宣布死亡情况

白天阶段：
1. 主持人宣布天亮
2. 死者留遗言
3. 存活玩家轮流发言
4. 投票放逐
5. 被放逐者留遗言
6. 判断狼人是否胜利
```

## 三、谈判场景

### 3.1 商业谈判

**场景描述**：模拟商务谈判，一方为买方，一方为卖方。

**角色配置**：
- 买方代表（Buyer）
- 卖方代表（Seller）
- 观察员（Observer，可选）

**规则配置**：
- 分轮报价
- 底线价格（不公开）
- 谈判时限
- 达成条件

**事件流**：
```
1. 双方获取各自底线和目标
2. 卖方先报价
3. 买方还价或接受
4. 双方多轮协商
5. 达成协议或谈判破裂
```

### 3.2 调解纠纷

**角色配置**：
- 调解员（Mediator）
- 甲方（Party A）
- 乙方（Party B）

**规则配置**：
- 各方先陈述诉求
- 调解员引导对话
- 不允许打断

## 七、核心交互流程

### 7.1 创建新讨论

```
用户操作                    系统响应
─────────────────────────────────────────────
1. 选择角色 ───────────────▶ 显示角色列表
2. 选择/创建角色 ──────────▶ 保存角色配置
3. 选择规则 ───────────────▶ 显示规则列表
4. 选择/创建规则 ──────────▶ 保存规则配置
5. 设置辩题 ───────────────▶ 保存辩题
6. 点击开始 ───────────────▶ 初始化会话，启动AI
7. 开始讨论 ◀────────────── 显示AI回复
```

### 7.2 实时讨论

```
用户输入                   系统处理                    AI响应
────────────────────────────────────────────────────────────────
用户发送消息 ────────────▶ 验证规则 ────────────────▶ 加载上下文
                            检查权限 ────────────────▶ AI决策
                            保存消息 ────────────────▶ 生成回复
                            触发心跳 ────────────────▶ 应用技能
                            广播消息 ◀─────────────── 显示AI回复
```

### 7.3 心跳主动发言

```
心跳触发                   系统处理                    AI行为
────────────────────────────────────────────────────────────────
定时器触发 ──────────────▶ 发布心跳事件 ───────────▶ 检查空闲
                            查询空闲AI ──────────────▶ 评估是否发言
                            检查规则约束 ────────────▶ 生成心理活动
                            验证发言条件 ────────────▶ 决定发言内容
                            广播消息 ◀─────────────── 显示AI主动发言
```
