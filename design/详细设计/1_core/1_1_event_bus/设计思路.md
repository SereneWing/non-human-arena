# 事件总线设计

## 一、设计目标

事件总线是整个系统模块间通信的核心枢纽，负责：
1. **解耦模块**：模块间不直接调用，通过事件通信
2. **统一通信**：所有事件通过统一的总线传递
3. **异步处理**：支持同步和异步事件处理
4. **可追踪**：记录所有事件便于调试

## 二、设计原则

1. **发布-订阅模式**：模块发布事件，订阅者处理事件
2. **类型安全**：事件类型枚举化，避免错误类型
3. **异步优先**：使用异步处理，不阻塞主线程
4. **可扩展**：支持本地和分布式实现

## 四、核心接口

```python
# 事件总线协议
class IEventBus(Protocol):
    """事件总线接口"""
    
    async def publish(self, event: Event) -> None:
        """发布事件"""
        ...
    
    async def subscribe(
        self,
        event_type: EventType,
        handler: Callable[[Event], Awaitable[None]]
    ) -> None:
        """订阅事件"""
        ...
    
    async def unsubscribe(
        self,
        event_type: EventType,
        handler: Callable[[Event], Awaitable[None]]
    ) -> None:
        """取消订阅"""
        ...
```

## 五、事件类型分类

| 类别 | 事件类型 | 说明 |
|------|---------|------|
| 角色 | ROLE_CREATED, ROLE_UPDATED, ROLE_DELETED | 角色生命周期 |
| 规则 | RULE_LOADED, RULE_TRIGGERED, RULE_VIOLATED | 规则相关 |
| 会话 | SESSION_CREATED, SESSION_STARTED, SESSION_PAUSED, SESSION_ENDED | 会话生命周期 |
| 消息 | MESSAGE_SENT, MESSAGE_RECEIVED | 消息流转 |
| AI | AI_THINK, AI_ACT, AI_SKIP, AI_HEARTBEAT | AI行为 |
| 系统 | HEARTBEAT_TICK, SYSTEM_SHUTDOWN | 系统事件 |

## 六、使用示例

```python
# 发布事件
await event_bus.publish(Event(
    type=EventType.MESSAGE_RECEIVED,
    source="user",
    session_id="session_123",
    data={"content": "Hello"}
))

# 订阅事件
async def on_message(event: Event):
    print(f"收到消息: {event.data}")

await event_bus.subscribe(EventType.MESSAGE_RECEIVED, on_message)
```
