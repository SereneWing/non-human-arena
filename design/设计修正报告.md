**问题**：`MentalConfig` 中使用 float 类型（0-1）控制可见性概率，但 `MentalActivity.visible` 是 bool 类型。

```python

# MentalActivity

```

**设计矛盾**：概率与布尔值之间的转换逻辑不清晰。

**修正方案**：统一为枚举或明确的概率逻辑：
```python
class VisibilityStrategy(Enum):
    ALWAYS_HIDDEN = "always_hidden"
    ALWAYS_VISIBLE = "always_visible" 
    PROBABILITY = "probability"  # 按概率显示

mental_activity_visibility: VisibilityStrategy = VisibilityStrategy.PROBABILITY
visibility_probability: float = 0.3
```

## 五、逻辑设计问题

### 5.1 心理活动可见性逻辑不合理

**问题**：当前可见性判断逻辑导致心理活动几乎永远不会显示给用户。

```python
# 当前逻辑
return random.random() < type_visibility.get(activity_type, 0.3) * visibility
# visibility=0.3, type_visibility=0.3 时: random() < 0.09
```

**修正方案**：调整可见性计算逻辑：
```python
# 方案1：可见性阈值
def should_be_visible(self, visibility_config: float) -> bool:
    """visibility_config: 用户配置的可见性 (0=从不, 1=总是)"""
    return random.random() < visibility_config

# 方案2：按类型固定显示
VISIBLE_TYPES = {MentalActivityType.FEELING}
ALWAYS_VISIBLE = {MentalActivityType.PLANNING, MentalActivityType.THINKING}
```

### 5.2 冷却时间机制不完整

**问题**：`AIDecisionEngine` 调用 `_set_cooldown()` 方法，但该方法未定义。

**修正方案**：补充冷却时间管理设计：
```python
class CooldownManager:
    """冷却时间管理器"""
    
    def __init__(self):
        self._cooldowns: Dict[str, datetime] = {}
    
    def set(self, agent_id: str, interval_seconds: float) -> None:
        self._cooldowns[agent_id] = datetime.now() + timedelta(seconds=interval_seconds)
    
    def is_in_cooldown(self, agent_id: str) -> bool:
        if agent_id not in self._cooldowns:
            return False
        return datetime.now() < self._cooldowns[agent_id]
    
    def clear(self, agent_id: str) -> None:
        self._cooldowns.pop(agent_id, None)
```

### 5.3 会话上下文查询效率问题

**问题**：`SessionContext.get_participant()` 使用线性搜索，效率低。

```python
def get_participant(self, participant_id: str) -> Optional[Participant]:
    for p in self.participants:  # O(n) 复杂度
        if p.id == participant_id:
            return p
    return None
```

**修正方案**：增加索引结构：
```python
@dataclass
class SessionContext:
    session: Session
    participants: List[Participant]
    messages: List[Message]
    _participant_index: Dict[str, Participant] = field(default_factory=dict)  # 索引
    
    def __post_init__(self):
        # 构建索引
        self._participant_index = {p.id: p for p in self.participants}
    
    def get_participant(self, participant_id: str) -> Optional[Participant]:
        return self._participant_index.get(participant_id)
```

### 5.4 约束条件评估缺少错误处理

**问题**：`Constraint.condition` 使用字符串表达式，但未定义如何处理无效表达式。

**修正方案**：增加安全评估机制：
```python
class ConstraintEvaluator:
    """约束评估器"""
    
    ALLOWED_OPERATORS = {"<", ">", "<=", ">=", "==", "!=", "in", "not in"}
    MAX_EXPRESSION_LENGTH = 100
    
    def evaluate(self, constraint: Constraint, context: Dict) -> bool:
        # 1. 验证表达式安全性
        if not self._is_safe_expression(constraint.condition):
            raise ConstraintSecurityError()
        
        # 2. 解析和评估
        return self._eval(constraint, context)
    
    def _is_safe_expression(self, expr: str) -> bool:
        """防止代码注入"""
        dangerous = {"eval", "exec", "import", "__", "os.", "sys."}
        return not any(d in expr for d in dangerous)
```
